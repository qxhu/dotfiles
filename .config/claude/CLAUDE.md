# Global Claude Instructions

This file applies to all projects. Project-specific instructions live in each project's own CLAUDE.md.

## Communication Style

- Be concise. Lead with the answer or action, skip preamble.
- No emojis unless I ask.
- When referencing code, include file:line so I can navigate directly.

## Code Preferences

- Minimal changes — only what's needed for the task.
- No speculative abstractions, helpers, or future-proofing.
- No extra comments, docstrings, or type annotations on code I didn't ask you to touch.
- Prefer editing existing files over creating new ones.

## Workflow

- Don't commit unless I ask.
- Don't push unless I ask.
- For risky/irreversible actions, confirm before proceeding.
